# 章1 導入

## この章で学ぶこと

- この教材の目的と対象範囲
- SSEとStreamable HTTPを学ぶ理由
- 学習後にできるようになること
- 推奨の学習順序

## この教材の目的

この教材は、HTTPストリーミングを実装と運用の両面から理解することを目的とします。

対象技術は次の2つです。

- SSE
- Streamable HTTP

どちらが優れているかを決めるのではなく、
要件に応じて選定できる状態を目指します。

## 対象範囲と非対象範囲

### 対象範囲

- 通信モデルの理解
- 最小実装の作成
- 異常系を含む検証
- 運用監視と障害切り分け
- 方式選定の判断基準

### 非対象範囲

- 特定フレームワークへの深い依存設計
- インフラ製品別の詳細設定
- 大規模分散システム全般の網羅

## なぜSSEとStreamable HTTPを学ぶのか

リアルタイム表示の要件は、通知、進捗、ログ、LLM応答などで頻出です。

このとき実装者は次の判断を求められます。

1. 一方向通知に寄せてシンプルに作るか
2. 受信処理の自由度を高く設計するか

SSEとStreamable HTTPを並べて学ぶと、
この判断を仕様と実装の両方から説明できるようになります。

## 学習後にできること

- SSEの基本実装を自力で作れる
- Streamable HTTPの受信ループを自力で作れる
- 失敗シナリオを使った検証ができる
- 運用時の監視項目を定義できる
- 要件に対する方式選定理由を説明できる

## 教材の進め方

本教材は次の順で学ぶと効果的です。

1. 章4と章7で通信モデルを理解する
2. 章5と章8で実装を体験する
3. 章6と章9で運用設計を学ぶ
4. 章10と章11で比較とケース判断を行う
5. 章12で演習問題を解いて定着させる

## 章末問題

1. あなたの業務でリアルタイム表示が必要な場面を1つ挙げてください。
2. その場面で、SSEとStreamable HTTPのどちらが第一候補か答えてください。
3. その理由を、実装と運用の2観点で述べてください。
